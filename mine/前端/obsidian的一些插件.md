# Obsidian 插件与工具整理

> [!abstract]
> 面向前端开发者的 Obsidian 使用清单。重点关注：==知识管理、代码笔记、工作流自动化==。

> [!tip] 我希望 Obsidian 帮我做到
> - 沉淀前端知识体系：HTML / CSS / JavaScript / TypeScript / React / 工程化
> - 整理源码阅读笔记、问题排查记录、最佳实践
> - 管理项目文档、学习路线、灵感与待办
> - 提高笔记自动化能力，减少重复整理成本

---

## 插件 / 工具关注清单

### 1. Claudian
- GitHub：[YishenTu/claudian](https://github.com/YishenTu/claudian)
- 定位：在 Obsidian 中接入 AI 助手，直接基于仓库 / 笔记上下文进行整理、分析和修改。

> [!info] 适合我的使用场景
> - 整理零散前端学习笔记
> - 把会议记录改写成结构化总结
> - 统一笔记格式、补标题、补小结
> - 辅助分析代码片段或项目文档

**我看重的点：**
- 能不能把临时记录整理成长期笔记
- 能不能围绕某个技术主题快速生成知识框架
- 能不能基于现有内容补充缺失信息

---

### 2. Obsidian Skills
- GitHub：[kepano/obsidian-skills](https://github.com/kepano/obsidian-skills)
- 定位：为 Obsidian / AI 工作流提供可复用能力模块。

> [!note]
> 这类工具更偏向“能力复用”和“流程标准化”，适合把重复性的知识整理动作模块化。

**我关注的问题：**
- 是否能和现有模板体系结合
- 是否适合前端知识库的分类方式
- 是否能减少重复整理成本

---

## 前端开发者更需要的 Obsidian 能力

### A. 代码与技术笔记
- 代码块高亮与可读性
- 支持记录源码分析、报错排查、性能优化案例
- 方便管理 JS / TS / React / 工程化相关主题

### B. 知识关联
- 能把「概念 → 问题 → 方案 → 踩坑记录」串起来
- 例如：
  - `React 性能优化`
  - `Vite 构建问题`
  - `TypeScript 类型体操`
  - `浏览器原理`

### C. 模板化输出
- 新建笔记时能快速套用模板，例如：
  - 问题排查模板
  - 源码阅读模板
  - 技术方案对比模板
  - 周报 / 复盘模板

### D. 自动化与 AI 辅助
- 自动总结
- 自动归类
- 自动补链接
- 基于上下文改写已有笔记

---

## 我筛选插件时的判断标准

| 维度 | 我会关注什么 |
| --- | --- |
| 是否适合前端 | 能否服务源码阅读、问题排查、技术沉淀 |
| 是否减少重复劳动 | 能否提升整理效率，而不是增加维护成本 |
| 是否方便长期积累 | 能否形成稳定的知识结构和可复用模板 |
| 是否易于集成 | 能否和现有笔记、标签、链接体系兼容 |
