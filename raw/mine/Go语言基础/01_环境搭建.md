## 一、安装 Go SDK

### macOS

```bash
# 推荐用 Homebrew 安装
brew install go

# 验证安装
go version
# 输出类似：go version go1.22.x darwin/arm64
```

### Windows

1. 下载安装包：[go.dev/dl](https://go.dev/dl/)
2. 双击安装，一路下一步
3. 打开 PowerShell，执行 `go version` 验证

### Linux

```bash
# 下载二进制包
wget https://go.dev/dl/go1.22.linux-amd64.tar.gz
sudo rm -rf /usr/local/go
sudo tar -C /usr/local -xzf go1.22.linux-amd64.tar.gz

# 把 /usr/local/go/bin 加入 PATH（写入 ~/.bashrc 或 ~/.zshrc）
export PATH=$PATH:/usr/local/go/bin

# 验证
go version
```

## 二、配置环境变量

Go 默认工作目录是 `$HOME/go`，可以通过 `GOPATH` 自定义：

```bash
# 查看当前配置
go env GOPATH
go env GOROOT
go env GOPROXY   # 模块代理，推荐用国内镜像

# 设置 GOPROXY（国内加速）
go env -w GOPROXY=https://goproxy.cn,direct
```

> [!tip] 常见 Go env 变量
> | 变量 | 作用 |
> |------|------|
> | `GOPATH` | 工作空间根目录（默认 `~/go`）|
> | `GOROOT` | Go 安装目录（一般不改）|
> | `GOPROXY` | 模块下载代理 |
> | `GO111MODULE` | 是否启用模块（`on`=模块模式，`auto`=自动）|

## 三、配置 VS Code

### 1. 安装插件

在 VS Code 扩展商店搜索 `Go`，安装由 **Go Team at Google** 出品的官方插件。

### 2. 安装工具链

打开命令面板（`Cmd/Ctrl + Shift + P`），输入 `Go: Install/Update Tools`，全选所有工具并安装：

| 工具            | 作用                         |
| ------------- | -------------------------- |
| `gopls`       | Language Server，代码补全/跳转/诊断 |
| `go-outline`  | 大纲视图                       |
| `gorename`    | 重命名重构                      |
| `delve`       | 调试器                        |
| `staticcheck` | 静态分析                       |

> [!warning] 第一次装插件后必须执行这一步
> 否则代码不会飘红（错误提示）、无法跳转、无法调试。

### 3. 推荐 VS Code 设置

```json
{
  "go.useLanguageServer": true,
  "go.formatTool": "gofmt",
  "go.lintTool": "staticcheck",
  "editor.formatOnSave": true
}
```

## 四、创建第一个程序

### 1. 创建项目目录

```bash
mkdir -p ~/go/src/hello
cd ~/go/src/hello
```

### 2. 初始化 Go 模块（必须）

```bash
go mod init hello
# 会在当前目录生成 go.mod 文件
```

### 3. 编写 Hello World

创建文件 `main.go`：

```go
package main

import "fmt"

func main() {
	fmt.Println("Hello, World!")
}
```

### 4. 运行

```bash
# 方式一：直接运行（编译临时文件并执行）
go run main.go

# 方式二：编译为可执行文件
go build -o hello main.go
./hello
```

> [!success] 验收标准
> 终端输出 `Hello, World!`，无任何报错。

## 六、常见问题

> [!faq]- go: command not found
> PATH 没配好，确认 `export PATH=$PATH:/usr/local/go/bin` 已写入 shell 配置文件，并执行 `source ~/.zshrc` 或重启终端。

> [!faq]- 插件装完不飘红 / 没有补全
> 执行 `Cmd/Ctrl + Shift + P` → `Go: Install/Update Tools` → 全选 → 回车，等待安装完成。

> [!faq]- go mod init 报错
> 确保当前目录是空目录，或者手动指定模块名：`go mod init github.com/你的用户名/项目名`

### 2. 配置私有模块访问

本项目使用阿里云私有模块仓库 `codeup.aliyun.com`，需配置环境变量：

```bash
# 配置私有模块路径
go env -w GOPRIVATE=codeup.aliyun.com
go env -w GONOPROXY=codeup.aliyun.com
go env -w GONOSUMDB=codeup.aliyun.com

# 验证
go env | grep -E 'GOPRIVATE|GONOPROXY|GONOSUMDB'
```

### 3. 配置 .netrc（凭据文件）

```bash
# 创建文件
touch ~/.netrc

# 写入凭据（用于访问 codeup.aliyun.com）
# 格式：machine {host} login {username} password {password}
machine codeup.aliyun.com login <用户名> password <密码>
```

> [!warning] 凭据安全
> `.netrc` 包含明文密码，确保文件权限正确：`chmod 600 ~/.netrc`

### 3.5 Go 依赖管理认证原理

> [!info] 参考文档
> [阿里云云效 - Go 依赖管理](https://help.aliyun.com/zh/yunxiao/user-guide/how-to-use-go-dependency-management)

Go 模块下载本质上是通过 HTTPS 请求私有仓库的代码。当模块路径被标记为私有（`GOPRIVATE`）时，`go` 命令不会走代理和校验，而是直接向对应 Git 服务器发起认证请求。

#### 认证流程

```
go get codeup.aliyun.com/xxx/module
        │
        ▼
检查 GOPRIVATE  → 匹配到私有路径，跳过代理和 sumdb
        │
        ▼
读取 ~/.netrc   → 找到 machine codeup.aliyun.com 对应的凭据
        │
        ▼
携带凭据发起 HTTPS 请求 → 下载模块源码
```

#### 关键环境变量

| 变量 | 作用 | 设置命令 |
|------|------|----------|
| `GOPRIVATE` | 标记私有模块，不走代理和校验 | `go env -w GOPRIVATE=codeup.aliyun.com` |
| `GONOPROXY` | 指定不通过代理下载的模块路径 | `go env -w GONOPROXY=codeup.aliyun.com` |
| `GONOSUMDB` | 指定不进行校验和查询的模块路径 | `go env -w GONOSUMDB=codeup.aliyun.com` |

> [!tip] 认证失败的常见原因
> - `.netrc` 文件权限不对（需要 `chmod 600`）
> - 凭据过期或账号密码错误
> - `GOPRIVATE` 未覆盖到实际的私有模块域名
> - 网络环境问题导致 HTTPS 请求超时

### 8. MySQL UI 工具

开发过程中需要一个数据库可视化管理工具来查看和调试数据，以下三款各有侧重：

| 工具 | 平台 | 特点 | 下载 |
|------|------|------|------|
| **DBeaver** | 全平台 | 功能最全面，支持多种数据库（MySQL、PostgreSQL、Redis 等），社区版免费 | [dbeaver.io/download](https://dbeaver.io/download/) |
| **Sequel Ace** | macOS | 专为 MySQL/MariaDB 设计，轻量快速，Mac App Store 免费 | [App Store](https://apps.apple.com/cn/app/sequel-ace/id1518036000?mt=12) |
| **TablePro** | 全平台 | 现代化界面，操作流畅 | [tablepro.app](https://tablepro.app/) |

> [!tip] 推荐选择
> - macOS 用户首选 **Sequel Ace**：App Store 一键安装，专做 MySQL，轻量不卡顿
> - 需要管理多种数据库选 **DBeaver**：一个工具覆盖所有数据源
> - 注重界面美观和交互体验选 **TablePro**

#### 安装 Sequel Ace（macOS 推荐）

直接在 Mac App Store 搜索 `Sequel Ace` 安装，或在终端执行：

```bash
# 通过 Homebrew 安装（备选方式）
brew install --cask sequel-ace
```

安装完成后，新建连接时需填写：
- **Host**：数据库服务器地址（本地开发填 `127.0.0.1`）
- **Port**：端口号（默认 `3306`）
- **Username / Password**：数据库账号密码

### 9. RPC 与 gRPC 调试工具

#### 什么是 RPC？

**RPC（Remote Procedure Call，远程过程调用）** 是一种通信协议，让程序像调用本地函数一样调用远程服务器上的方法，底层网络通信对调用者透明。

```
客户端                         服务端
  │                              │
  ├── 调用本地函数 stu.GetName()  │
  │         │                    │
  │    RPC 框架封装请求           │
  │         │──── 网络传输 ────→  │
  │         │                    ├── 执行 GetName()
  │         │←─── 返回结果 ──────  │
  │         │                    │
  ├── 拿到结果，继续执行          │
```

**gRPC** 是 Google 开源的高性能 RPC 框架，使用 **Protocol Buffers**（protobuf）作为接口描述语言和序列化格式，是 Go 微服务之间通信的主流方案。

#### 安装 BloomRPC

[BloomRPC](https://github.com/bloomrpc/bloomrpc) 是一款 gRPC 接口调试工具，类似于 REST API 的 Postman，可以直接导入 `.proto` 文件并发送请求。

**macOS：**

```bash
brew install --cask bloomrpc
```

**其他平台：**

前往 [GitHub Releases](https://github.com/bloomrpc/bloomrpc/releases) 下载对应平台的安装包。

#### 使用方式

1. 打开 BloomRPC，点击左上角 `+` 按钮
2. 导入项目中的 `.proto` 文件
3. 在左侧选择要调试的 Service 和 Method
4. 填写请求参数（JSON 格式），设置服务地址（如 `localhost:8087`）
5. 点击发送，查看响应结果

> [!tip] proto 文件在哪
> 项目中的 `.proto` 文件通常位于服务端的 `api/` 或 `proto/` 目录下，具体路径可询问后端同事。

> [!faq]- BloomRPC 连接失败
> 确认 gRPC 服务已启动，且地址和端口填写正确（如 `localhost:8087`）。gRPC 默认使用 HTTP/2，确保没有被代理或防火墙拦截。

> [!faq]- 无法解析响应
> 检查导入的 `.proto` 文件是否与服务端使用的版本一致，proto 定义变更需重新导入。
