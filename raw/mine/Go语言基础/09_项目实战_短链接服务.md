# 项目实战：短链接服务

> [!abstract] 项目目标
> 构建一个短链接服务：长 URL → 短码 → 重定向到原 URL。
> 练习点：Gin 路由、Path 参数、301 重定向、内存存储。

---

## 一、项目需求

| 功能 | 描述 |
|------|------|
| 生成短链接 | 传入长 URL，返回短码 |
| 短链接访问 | 访问 `/:code` 时重定向到原 URL |
| 短码查重 | 相同的 URL 返回已有的短码，不重复生成 |

### URL 设计

```
POST /api/shorten    →  {"url": "https://xxx.com/xxx"}  →  {"short_url": "/abc123", "code": "abc123"}
GET  /:code          →  301 重定向到原 URL
GET  /api/url/:code  →  查看某个短码的原始 URL（调试用）
```

## 二、完整代码

### go.mod

```bash
go mod init shorturl
go get github.com/gin-gonic/gin
```

### main.go

```go
package main

import (
    "crypto/sha256"
    "encoding/base64"
    "encoding/json"
    "fmt"
    "net/http"
    "sync"
    "time"

    "github.com/gin-gonic/gin"
)

// ===== 模型 =====

type ShortURL struct {
    Code     string    `json:"code"`
    Original string    `json:"original"`
    CreatedAt time.Time `json:"created_at"`
}

type ShortenRequest struct {
    URL string `json:"url" binding:"required,url"`
}

type ShortenResponse struct {
    ShortURL string `json:"short_url"`
    Code     string `json:"code"`
    Original string `json:"original"`
}

// ===== 存储 =====

var (
    codeMap  = make(map[string]ShortURL)  // code → ShortURL
    urlMap   = make(map[string]string)     // originalURL → code（用于查重）
    mu       sync.RWMutex
)

// ===== 工具函数 =====

// 生成短码：取 URL 的 SHA256 前 6 位，base64 编码
func generateCode(url string) string {
    hash := sha256.Sum256([]byte(url))
    return base64.URLEncoding.EncodeToString(hash[:])[:6]
}

// ===== Handler =====

// POST /api/shorten — 创建短链接
func shorten(c *gin.Context) {
    var req ShortenRequest
    if err := c.ShouldBindJSON(&req); err != nil {
        c.JSON(http.StatusBadRequest, gin.H{"error": "无效的 URL"})
        return
    }

    mu.Lock()
    defer mu.Unlock()

    // 查重：已经生成过的 URL 不重复生成
    if code, exists := urlMap[req.URL]; exists {
        base := c.Request.Host
        c.JSON(http.StatusOK, ShortenResponse{
            ShortURL: fmt.Sprintf("http://%s/%s", base, code),
            Code:     code,
            Original: req.URL,
        })
        return
    }

    // 生成新的短码
    code := generateCode(req.URL)
    short := ShortURL{
        Code:     code,
        Original: req.URL,
        CreatedAt: time.Now(),
    }

    codeMap[code] = short
    urlMap[req.URL] = code

    base := c.Request.Host
    c.JSON(http.StatusCreated, ShortenResponse{
        ShortURL: fmt.Sprintf("http://%s/%s", base, code),
        Code:     code,
        Original: req.URL,
    })
}

// GET /api/url/:code — 查看原始 URL（调试用）
func getOriginal(c *gin.Context) {
    code := c.Param("code")

    mu.RLock()
    short, exists := codeMap[code]
    mu.RUnlock()

    if !exists {
        c.JSON(http.StatusNotFound, gin.H{"error": "短码不存在"})
        return
    }

    c.JSON(http.StatusOK, gin.H{
        "code":     short.Code,
        "original": short.Original,
        "created":  short.CreatedAt,
    })
}

// ===== main =====

func main() {
    r := gin.Default()

    // API 路由
    api := r.Group("/api")
    {
        api.POST("/shorten", shorten)
        api.GET("/url/:code", getOriginal)
    }

    // 短码路由（必须放在 /api 路由之后，避免冲突）
    // GET /abc123 → 重定向
    r.GET("/:code", func(c *gin.Context) {
        code := c.Param("code")

        mu.RLock()
        short, exists := codeMap[code]
        mu.RUnlock()

        if !exists {
            c.JSON(http.StatusNotFound, gin.H{"error": "短码不存在"})
            return
        }

        // 301 永久重定向
        c.Redirect(http.StatusMovedPermanently, short.Original)
    })

    fmt.Println("短链接服务启动: http://localhost:8080")
    r.Run(":8080")
}
```

## 三、测试

### 1. 创建短链接

```bash
curl -X POST http://localhost:8080/api/shorten \
  -H "Content-Type: application/json" \
  -d '{"url": "https://github.com/gin-gonic/gin"}'

# 返回：
# {
#   "short_url": "http://localhost:8080/V1u6Q=",
#   "code": "V1u6Q=",
#   "original": "https://github.com/gin-gonic/gin"
# }
```

### 2. 访问短链接（浏览器测试）

在浏览器打开 `http://localhost:8080/V1u6Q=`，会自动 301 重定向到 GitHub。

### 3. 查看原始 URL

```bash
curl http://localhost:8080/api/url/V1u6Q=
# {"code":"V1u6Q=","original":"https://github.com/gin-gonic/gin","created":"2026-04-16T10:30:00Z"}
```

### 4. 查重验证

再次调用 `POST /api/shorten` 相同的 URL，会返回**同一个短码**：

```bash
curl -X POST http://localhost:8080/api/shorten \
  -H "Content-Type: application/json" \
  -d '{"url": "https://github.com/gin-gonic/gin"}'

# 返回相同的 code，不会重复生成
```

## 四、核心知识点解析

### 301 vs 302 重定向

| 状态码 | 类型 | 适用场景 |
|--------|------|---------|
| `301` | 永久重定向 | 短链接（告诉浏览器/搜索引擎这是永久地址）|
| `302` | 临时重定向 | 临时活动页、A/B 测试 |
| `307` | 临时重定向（保持 Method）| API 跳转 |

```go
c.Redirect(http.StatusMovedPermanently, short.Original)  // 301
c.Redirect(http.StatusFound, short.Original)              // 302
```

### 路由冲突处理

```go
// ⚠️ 错误：/api/:code 会匹配 /api/shorten
api.GET("/url/:code", ...)  // 先注册具体路由

// 然后再注册泛型路由
r.GET("/:code", ...)         // 后注册兜底路由
```

> [!warning] 路由注册顺序很重要
> Gin 按注册顺序匹配路由。先注册 `/api/url/:code`，后注册 `/:code`。
> 如果反过来，`/:code` 会把 `/api/shorten` 也匹配掉！

## 五、后续扩展方向

| 方向 | 说明 |
|------|------|
| 持久化 | 存储到 Redis 或 MySQL，重启不丢失 |
| 有效期 | 给 ShortURL 加 `expire_at` 字段，过期返回 410 |
| 自定义短码 | 支持用户指定短码 `POST /api/shorten?code=mycode` |
| 访问统计 | 加 `visitCount` 字段，每次访问 +1 |
| 布隆过滤器 | URL 很多时，用布隆过滤器快速判断是否存在 |
