# Go 语言学习

## 🎯 学习目标

> [!abstract] 目标概览
> 作为前端开发，学习 Go 的核心目标是：**补齐后端知识短板，打通全栈协作最后一公里。**

- [ ] 学完 Go 基础语法，能读懂简单 Go 代码，完成基础语法学习并整理笔记
- [ ] 能看懂项目中与前端相关的接口结构、入参出参含义，理清前后端交互逻辑
- [ ] 结合 AI 辅助理解复杂服务端逻辑，尝试用 AI 辅助编写简单接口示例，逐步具备基础全栈排查与简单协作开发能力

## 📋 阶段规划

### 阶段一：开发环境搭建

详细步骤见 → [[01_环境搭建]]

| 工具 | 作用 | 验证命令 |
|------|------|----------|
| Go SDK | 编译器与工具链 | `go version` |
| VS Code + Go 插件 | 编辑器 + 补全/调试 | — |
| curl / Postman | 接口调试 | — |
| Git | 版本控制 | `git --version` |

> [!tip] 第一天行动
> 安装 Go SDK 后，写一个 `hello.go`，用 `go run hello.go` 跑通，消除对命令行的陌生感。

### 阶段二：基础语法（建议 2 周）

详细笔记：[[02_基础语法速查]] / [[03_数据类型]] / [[04_函数与错误处理]] / [[05_结构体与接口]]

```
Week 1
├── 变量与数据类型（int, string, bool, slice, map）
├── 控制流（if / for / switch）
├── 函数（多返回值、错误处理）
└── 练习：写一个计算器

Week 2
├── 结构体（struct）与方法
├── 接口（interface）— 理解前端没有的概念
├── 包（package）与模块（module）
├── goroutine + channel（了解概念即可）
└── 练习：写一个内存版 CRUD 接口
```

### 阶段三：前后端交互（核心目标）⭐

这是前端开发者最需要补的短板，详细笔记：[[06_HTTP与接口基础]] / [[07_Gin框架入门]]

重点掌握：

| 概念 | 关键点 |
|------|--------|
| HTTP 协议 | 请求方法、状态码、Header、JSON Body |
| RESTful API | GET / POST / PUT / DELETE 语义 |
| JSON 编解码 | `json.Marshal` / `json.Unmarshal` |
| 请求解析 | 读 Body、Query 参数、Path 参数 |
| 响应构造 | 设置状态码 + 写 JSON 响应 |

> [!example] 最小闭环示例
> 理解这个最小接口，前后就打通了：
> ```go
> func handler(w http.ResponseWriter, r *http.Request) {
>     json.NewEncoder(w).Encode(map[string]string{"message": "Hello"})
> }
> http.ListenAndServe(":8080", nil)
> ```
> 用 Postman 或前端 `fetch` 调一下，感受 HTTP 的双向交互。
