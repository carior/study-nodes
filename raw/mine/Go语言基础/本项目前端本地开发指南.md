# 本项目前端本地开发指南

> [!info] 用途说明
> 本笔记记录本项目（drsp）前端本地开发时的代理配置，使前端能连接本地 Go 服务进行联调。

---

## 一、目标

让前端项目在开发模式下，将请求代理到：
- `/api/api/copyright` → 本地 Go 服务 `http://127.0.0.1:8087`
- 其他 `/api/*` → 线上生产环境 `https://drsp.qmniu.com`

---

## 二、Vite 代理配置

文件路径：`apps/drsp.qmniu.com/vite.config.ts`

```ts
server: {
    ...(_common as any).server,
    proxy: {
        // /api/api/* 转发到本地 Go 服务，去掉多余的 /api 前缀
        '/api/api/copyright': {
            target: 'http://127.0.0.1:8087',
            changeOrigin: true,
            rewrite: (path) => path.replace(/^\/api\/api\/copyright/, '/api/copyright'),
        },
        // /api/* 转发到线上
        '/api': {
            target: 'https://drsp.qmniu.com',
            changeOrigin: true,
            rewrite: (path) => path.replace(/^\/api/, ''),
        },
    },
},
```

### 配置解析

| 配置项 | 含义 |
|--------|------|
| `target` | 后端服务地址 |
| `changeOrigin: true` | 修改请求头中的 Origin，适配后端 CORS |
| `rewrite` | 重写路径（本项目路径多了一层 `/api`，需去掉）|

> [!tip] 为什么要 rewrite？
> 前端请求 `/api/api/copyright`，但本地 Go 服务的路径是 `/api/copyright`。
> `rewrite` 把多余的 `/api` 前缀去掉后转发。

## 三、环境变量配置

文件路径：`apps/drsp.qmniu.com/.env.development`

```bash
# 注释掉默认的代理配置（否则会冲突）
# VITE_PROXY = [["/api","https://drsp.qmniu.com"]]
```

> [!warning] 注释原因
> 默认代理会拦截所有 `/api` 请求并转发到线上，与本地开发代理冲突。
> 取消注释后本地代理配置失效。

## 五、启动顺序

```bash
# 1. 先启动本地 Go 服务（见 [[01_环境搭建]]）
go run main.go http

# 2. 再启动前端开发服务器
cd apps/drsp.qmniu.com
pnpm dev
```

> [!success] 验证方法
> 打开浏览器控制台 Network 面板，确认：
> - `/api/api/copyright` 请求 → 状态 200，目标是 `127.0.0.1:8087`
> - 其他 `/api/*` 请求 → 目标是 `drsp.qmniu.com`
