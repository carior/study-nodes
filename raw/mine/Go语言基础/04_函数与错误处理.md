# Go 函数与错误处理

> [!info] 章节说明
> Go 的函数设计有两个独特之处：**多返回值**和**显式错误处理**。这与前端的 `try-catch` 有本质区别，是 Go 编程的核心习惯。

---

## 一、函数基础

### 标准函数

```go
func greet(name string) string {
    return "Hello, " + name
}

msg := greet("张三")
fmt.Println(msg) // Hello, 张三
```

### 多返回值（Go 特色）

Go 函数可以返回多个值，最常用的场景是**返回结果 + 错误**：

```go
func divide(a, b int) (int, error) {
    if b == 0 {
        return 0, errors.New("除数不能为零")
    }
    return a / b, nil  // nil 表示没有错误
}
```

> [!tip] 为什么 Go 用返回值传错误？
> - 显式：每个调用都必须显式处理错误，不处理会编译报错
> - 简单：不需要 `try-catch-finally` 嵌套
> - 性能：不引入额外的异常机制开销

---

## 二、错误处理模式

### 基本模式

```go
result, err := divide(10, 2)
if err != nil {
    // 处理错误
    fmt.Println("出错了:", err)
    return  // 或 return err
}
fmt.Println("结果:", result)
```

### 简化常见写法

```go
// 不需要 result 时，用 _ 忽略
_, err := divide(10, 0)
if err != nil {
    fmt.Println(err)
}
```

### 错误链（errors.Wrap）

```go
import "fmt"

// 包装错误，添加上下文
if err != nil {
    return fmt.Errorf("读取配置失败: %w", err)
}
```

> [!example] 对比前端
> | 前端 JS | Go |
> |---------|-----|
> | `try { fn() } catch(e) { ... }` | `result, err := fn(); if err != nil { ... }` |
> | `throw new Error()` | `return nil, errors.New("msg")` |
> | `finally { ... }` | `defer { ... }` |

## 三、defer（延迟执行）

`defer` 关键字让函数在**退出前**执行，常用于资源清理：

```go
func readFile(name string) {
    // defer 保证文件一定被关闭，无论下面代码是否报错
    file, err := os.Open(name)
    if err != nil {
        return
    }
    defer file.Close()  // 在 readFile 返回前执行

    // 读取文件...
}
```

> [!tip] defer 执行顺序
> 后进先出（类似栈）：
> ```go
> defer fmt.Println("1")
> defer fmt.Println("2")
> defer fmt.Println("3")
> // 函数结束时输出顺序：3 → 2 → 1
> ```

## 四、函数类型与回调

```go
// 函数作为参数
func apply(fn func(int) int, n int) int {
    return fn(n)
}

result := apply(func(x int) int {
    return x * 2
}, 5)
fmt.Println(result) // 10
```

> [!tip] 匿名函数
> 类似 JS 的箭头函数：`(func(x int) int { return x * x })(5)` 直接调用。

## 五、变长参数

```go
func sum(nums ...int) int {
    total := 0
    for _, n := range nums {
        total += n
    }
    return total
}

fmt.Println(sum(1, 2, 3))        // 6
fmt.Println(sum(1, 2, 3, 4, 5))  // 15
```

## 六、命名返回值

```go
func split(sum int) (x, y int) {
    x = sum * 4 / 9
    y = sum - x
    return // 返回命名变量（裸 return）
}
```

> [!warning] 命名返回值慎用
> 只在简短函数中使用，复杂逻辑中用显式 `return result` 更清晰。

## 七、闭包

```go
// 计数器工厂
func counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

next := counter()
fmt.Println(next()) // 1
fmt.Println(next()) // 2
fmt.Println(next()) // 3
```

## 八、panic 与 recover（异常处理）

> [!warning] 谨慎使用
> Go 的 `panic` 类似前端的 `throw`，但**极少使用**。Go 的哲学是"用 error 返回错误"，只有真正不可恢复的程序错误才用 panic。

```go
func safeCall() {
    defer func() {
        if r := recover(); r != nil {
            fmt.Println("捕获了 panic:", r)
        }
    }()

    panic("Oops!")
    fmt.Println("不会执行到这里")
}

safeCall() // 输出：捕获了 panic: Oops!
```

## 九、HTTP 接口中的错误处理

服务端最常见的错误处理模式：

```go
func handleRequest(w http.ResponseWriter, r *http.Request) {
    // 1. 解析请求
    var req Request
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        // 返回 400 Bad Request
        http.Error(w, `{"error": "请求格式错误"}`, http.StatusBadRequest)
        return
    }

    // 2. 业务逻辑
    result, err := process(req)
    if err != nil {
        // 返回 500 Internal Server Error
        http.Error(w, `{"error": "`+err.Error()+`"}`, http.StatusInternalServerError)
        return
    }

    // 3. 返回结果
    json.NewEncoder(w).Encode(result)
}
```

> [!success] 核心模式
> 处理错误 → 返回 HTTP 错误码 → 直接 `return`，避免继续执行。
