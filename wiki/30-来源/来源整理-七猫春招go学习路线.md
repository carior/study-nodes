原文在多个阶段都插入了“练习”条目，并说明“题目要求见文件，开始做练习时再进行细致讲解”。  
在整理版中，这些练习统一理解为：

- 阶段性巩固任务
- 与当前模块绑定的实操题
- 需要在看完内容后立即动手完成

> [!note] 练习的作用
> 这份路线不是“看完就行”，而是“看懂 + 做出来 + 能讲清楚为什么这么做”。

## 给前端开发者的学习顺序建议

如果你是前端出身、还没有服务端基础，建议优先顺序改成：

1. **HTTP / DNS / 请求响应 / 状态码**
2. **Go 基础语法 + `net/http`**
3. **Gin + Postman + Charles**
4. **MySQL + Redis**
5. **Git / Linux / go module / 工具链**
6. **RabbitMQ / MongoDB / Nginx / CDN**

> [!important] 前端转后端的关键点
> 先把“前后端怎么通信”学透，再补语法和中间件，会比从头啃语法更容易建立成就感。
