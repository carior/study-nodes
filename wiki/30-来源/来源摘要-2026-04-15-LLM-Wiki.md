## 一句话总结

用 LLM 替代人类维护者，构建一个"知识编译一次、持续复利积累"的个人 Wiki，区别于传统 RAG 的"每次从零推导"。

---

## 核心观点

### 1. 关键洞察：Wiki 是持久复利的产物

传统 RAG = 每次查询重新发现知识 → 无积累

LLM Wiki = 新来源 → LLM 更新相关页面（可能 10-15 个）→ 交叉引用已建好、矛盾已标记 → **知识持续复利**

### 2. 三层架构

| 层级 | 说明 |
|------|------|
| **Raw Sources** | 原始文档，只读不修改 |
| **Wiki** | LLM 全权维护的摘要/主题/实体/比较层 |
| **Schema** | 约定结构、工作流和维护规则 |

### 3. 三种操作

| 操作 | 描述 |
|------|------|
| **Ingest** | 新来源 → 摘要 → 更新相关页 → 更新 index/log |
| **Query** | 先读 index → 主题/实体页 → 综合回答 → 主动沉淀高价值输出 |
| **Lint** | 健康检查：孤儿页、矛盾、过时结论、数据缺口 |

### 4. 两个关键文件

- **`index.md`**：内容导向，按页面类型列出所有页面，回答问题前先读它
- **`log.md`**：时间导向，只追加不修改历史，格式 `## [YYYY-MM-DD] 动作 | 标题`

### 5. 可选 CLI 工具建议

| 工具 | 作用 |
|------|------|
| **qmd** | 本地 Markdown 搜索（BM25/向量搜索 + LLM 重排），有 CLI 和 MCP 服务 |
| **Obsidian Web Clipper** | 浏览器插件，将网页文章转为 Markdown |
| **图片本地化** | 设置固定附件目录 + 快捷键下载图片，LLM 可直接读取本地图片 |
| **Graph View** | Obsidian 内置，查看知识库结构：哪些页面是 Hub，哪些是孤儿 |
| **Marp** | Markdown 幻灯片格式，可从 wiki 内容生成演示 |
| **Dataview** | Obsidian 插件，通过 frontmatter 属性查询生成动态列表和表格 |
