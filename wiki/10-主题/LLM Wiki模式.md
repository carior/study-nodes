# LLM Wiki 模式

> [!abstract] 核心定义
> LLM Wiki 是一种由 LLM 持续增量维护的个人知识库模式。其核心思想是：知识**编译一次，持续保持最新**，而不是每次查询时从原始文档重新推导。

## 核心思想

传统 RAG（检索增强生成）的问题：

- 每次提问时，LLM 从原始文档中重新发现知识
- 问一个需要综合 5 份文档的问题，LLM 必须每次都重新找、重新拼凑
- 没有积累，没有增量

LLM Wiki 的解决方案：

- 在**原始资料**和**使用者**之间，插入一个 LLM 持续更新的 Wiki 层
- 新来源加入时，LLM 读取后**主动更新相关页面**（主题页、实体页、索引）
- Wiki 本身是**持久复利的知识产物**，交叉引用已建好，矛盾已标记，结论已沉淀

> [!quote]
> "你几乎从不自己写 wiki——LLM 替你写、替你维护。你负责的是来源、探索和提问。LLM 负责所有繁琐的归纳、交叉引用、整理和记录工作。"
> — Andrej Karpathy

## 三层架构

```
┌────────────────────────────────────┐
│         Raw Sources（只读）          │
│   原始文档、截图、会议记录、网页剪藏   │
│   永不覆盖、永不改写                  │
└────────────────┬───────────────────┘
                 │ LLM 读取
                 ▼
┌────────────────────────────────────┐
│         Wiki（LLM 维护）             │
│   摘要、主题页、实体页、比较页、索引   │
│   用户读这一层，LLM 写这一层          │
└────────────────┬───────────────────┘
                 │ LLM 按 Schema 规则维护
                 ▼
┌────────────────────────────────────┐
│         Schema（AGENTS.md）          │
│   目录规范、工作流规则、页面约定        │
└────────────────────────────────────┘
```

## 三种操作

### 1. Ingest（摄取）

将新来源加入知识库：

```
读取来源 → 产出来源摘要 → 更新相关主题页/实体页 → 更新 index → 追加 log
```

一篇来源可能影响 **10-15 个** wiki 页面。

> [!tip] 推荐节奏
> 每次只摄取 1 个来源，边摄取边读摘要、引导 LLM 强调重点。积累一段时间后可批量处理。

### 2. Query（查询）

向知识库提问：

```
wiki/index → 相关主题/实体页 → 综合回答 → 主动建议沉淀高价值结论
```

> [!tip] 高价值输出 → 沉淀回 wiki
> 一次对比分析、一次有洞见的连接、一次阶段性结论——这些不应该消失在聊天历史里，应该写回 wiki。

### 3. Lint（健康检查）

定期检查知识库健康度：

- 孤儿页（没有入链的页面）
- 高频出现但未建页的概念/实体
- 页面间互相矛盾的结论
- 被新来源修正的旧结论
- 散落在别处的来源摘要内容
- `index.md` 是否遗漏新页面
- `log.md` 是否缺少关键记录

## 适用场景

| 场景 | 例子 |
|------|------|
| 个人成长 | 追踪目标、健康、心理、自我提升 |
| 深度研究 | 数周/月内深入一个主题，阅读论文并构建论点 |
| 读书笔记 | 按章节归档，建立人物/主题/情节的知识库（类似 Tolkien Gateway）|
| 团队知识库 | LLM 维护 Slack、会议记录、项目文档的人类辅助 wiki |
| 竞品分析/尽职调查/旅行规划/课程笔记 | 任何需要时间积累的知识领域 |

## 为什么有效

> [!success]
> 维护知识库最累人的部分不是阅读或思考，而是**整理**：更新交叉引用、保持摘要最新、标记新旧矛盾、维持多页面一致性。人类放弃 wiki 是因为维护负担增长比价值更快。LLM 不会厌倦、不会忘记更新交叉引用，一次能改 15 个文件。维护成本接近零，wiki 就能持续维护。

人类的工作：筛选来源、引导分析、提问、判断意义
LLM 的工作：所有整理工作

这与 Vannevar Bush 的 Memex（1945）精神相近——个人策划的知识库，文档间有联想路径。Bush 无法解决的是"谁来维护"。LLM 解决了这个问题。
